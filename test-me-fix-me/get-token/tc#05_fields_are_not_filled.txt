Тест-кейс 5: Поля не заполнены.

Цель: убедиться, что при отправке некорректного POST запроса, с не заполненными полями, от сервера приходит сообщение об ошибке.

Шаги: 
1. Отправить некорректный POST запрос, с не заполненными полями на /get-token.

Например,

{
  "username": "",
  "pwd": "",
  "email": ""
}

2. 	Получена ошибка от сервера, что поля должны быть корректно заполнены.

Ожидаемый результат: 
Ответ от сервера вида:

{
    "timestamp": "10-10-2024 01:09:27",
    "status": "ERROR",
    "message": "username must not be empty"
}
