Тест-кейс 4: Пропущено поле username.

Цель: убедиться, что при отправке некорректного POST запроса, с пропущенным полем username, от сервера приходит сообщение об ошибке.

Шаги: 
1. Отправить некорректный POST запрос, с пропущенным полем username на /get-token.

Например,

{
  "pwd": "validpassword",
  "email": "valid@example.com"
}
2. 	Получена ошибка от сервера, что username не может быть пустым.

Ожидаемый результат: 
Ответ от сервера вида:

{
    "timestamp": "10-10-2024 01:09:27",
    "status": "ERROR",
    "message": "username must not be empty"
}
