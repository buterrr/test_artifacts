Тест-кейс 06: Установка значения, содержащего некорректный тип данных (строка вместо числа).

Цель: убедиться, что на попытку установления, значения, содержащего некорректный тип данных (строка вместо числа),
от приложения приходит ответ с ошибкой 400 и сообщением, что передан некорректный тип данных.

 
Шаги: 

1. Отправить запрос на метод /set-amount содержащий некорректный тип данных (строка вместо числа).
(Например, value = 'abc')
2. Получен ответ, содержащий ошибку 400 и сообщение, что передан некорректный тип данных.

Ожидаемый результат: Ответ от сервера вида:

{
    "timestamp": "14-10-2024 12:51:05",
    "status": "ERROR",
    "message": "Передан некорректный тип входных данных"
}
